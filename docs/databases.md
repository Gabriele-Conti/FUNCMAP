# FUNCMAP database configuration

FUNCMAP uses a database sheet to define external database paths.

The database sheet is passed to the pipeline using:

```bash
--dbsheet /path/to/databases_funcmap.csv
```

## Database sheet format

The database sheet is a CSV file with the following columns:

```csv
tool,db_name,db_path
humann,chocophlan,/path/to/humann/chocophlan
humann,uniref90,/path/to/humann/uniref90
humann,full_mapping_v201901b,/path/to/humann/full_mapping_v201901b
rgi,CARD,/path/to/preloaded/CARD_RGI
```

## Columns

| Column | Description |
|---|---|
| `tool` | Tool or workflow branch using the database. Supported values: `humann`, `rgi`. |
| `db_name` | Database identifier expected by the pipeline modules. |
| `db_path` | Path to the database directory. |

FUNCMAP does not currently download or prepare external databases automatically. All database paths must point to pre-existing database directories.

## Required database entries

### HUMAnN

FUNCMAP expects the following HUMAnN database entries:

```csv
humann,chocophlan,/path/to/humann/chocophlan
humann,uniref90,/path/to/humann/uniref90
humann,full_mapping_v201901b,/path/to/humann/full_mapping_v201901b
```

The `full_mapping_v201901b` directory should contain the mapping files required for regrouping and renaming HUMAnN outputs, including KEGG, CAZy, Pfam and eggNOG mappings.

### RGI/CARD

FUNCMAP expects a preloaded local CARD/RGI database directory:

```csv
rgi,CARD,/path/to/preloaded/CARD_RGI
```

The CARD/RGI directory should contain the local RGI database, including the `localDB/` directory generated by RGI database loading commands such as:

```bash
rgi auto_load --local
```

## Path recommendations

Use absolute paths whenever possible.

Recommended:

```csv
rgi,CARD,/scratch/databases/CARD_RGI
```

Avoid relying on relative paths for production runs, especially on HPC systems.

## Container bind visibility

All database paths must be visible inside the container.

When using Apptainer, make sure parent directories are bind-mounted. With:

```groovy
apptainer.autoMounts = true
```

Nextflow usually handles input files and work directory mounts automatically. However, database locations outside standard project/work directories may still require cluster-specific bind configuration.

If a database cannot be found at runtime, check:

1. the path exists on the host system;
2. the path is readable by the user running Nextflow;
3. the path is visible inside the Apptainer container;
4. the path in `databases_funcmap.csv` matches the expected mounted path.

For example, on systems where databases are stored under `/scratch` and `/dati`, Apptainer bind options may be passed through the pipeline parameter:

```bash
--apptainer_run_options '-B /scratch:/scratch -B /dati:/dati'
```
